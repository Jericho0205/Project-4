# Deployment Process

## Updated Deployment Workflow

Because Vercel no longer supports the manual upload feature and GitHub Classroom repositories cannot connect directly to Vercel, I followed the required workaround.

## Steps Followed

1. Completed the project in my private GitHub Classroom repository.
2. Committed and pushed all source code to the Classroom repository.
3. Created a private fork of the GitHub Classroom repository.
4. Added the instructor’s GitHub account, `christoforou`, as a collaborator with read access.
5. Logged into Vercel.
6. Selected “Add New Project.”
7. Connected my GitHub account.
8. Selected the private forked repository.
9. Deployed the project on Vercel.
10. Tested the Vercel deployment URL.

## Issues Encountered

The original GitHub Classroom repository could not be connected directly to Vercel, so I used a private fork.
